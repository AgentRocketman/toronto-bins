📡 API ENDPOINT

Endpoint: POST https://agentrocketman.com/api/send-completion-email

Required Fields:
  • address (string) - Customer service address
  • serviceType (string) - "roll-out" or "roll-in"
  • workerName (string) - Service worker name
  • completedDateTime (string, ISO 8601) - Completion timestamp

Optional Fields:
  • stopId - Service stop reference
  • completed - Completion status boolean
  • imageUrl - Completion photo URL

Success Response (200):
  {
    "success": true,
    "emailSent": true,
    "to": "customer@example.com",
    "messageId": "..."
  }

Error Response (400/404/500):
  {
    "success": false,
    "emailSent": false,
    "error": "Description of what went wrong"
  }

🎯 WHAT THIS DOES

When your CurbIn app calls the endpoint:

  POST /api/send-completion-email
  {
    "address": "123 Main St, Toronto",
    "serviceType": "roll-out",
    "workerName": "John Smith",
    "completedDateTime": "2026-06-14T15:30:00Z",
    "imageUrl": "https://example.com/photo.jpg"
  }

The backend:

  1. Validates all input
  2. Looks up customer in Airtable by address
  3. Generates beautiful HTML email with:
     - Service address
     - Service type (Roll Out / Roll In)
     - Worker name  
     - Completion time (Toronto timezone)
     - Completion photo (if provided)
  4. Sends email via Hostinger SMTP
  5. Returns success/failure response

📞 SUPPORT

Issues? Check these:

  • Verify npm: npm --version

Email not sending:

  • Check Hostinger account is active
  • Run: pm2 logs curbin-email

Customer not found:
  • Check address matches Airtable exactly
