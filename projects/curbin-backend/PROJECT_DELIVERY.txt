📊 API ENDPOINTS (7 TOTAL)

1. GET /api/health
   Purpose: Server health check
   Response: { status: "OK", timestamp: "..." }

2. GET /api/test-airtable
   Purpose: Create 8 sample service stops
   Response: 8 records with all fields

3. GET /api/services
   Purpose: Retrieve all services from Airtable
   Response: Array of service stops with IDs

4. POST /api/upload
   Purpose: Upload service image
   Accepts: multipart/form-data (file, stopId, date)
   Response: { success, imageUrl, filename, size }

5. POST /api/save-service
   Purpose: Save service stop to Airtable
   Body: { id, address, type, date, completed, imageUrl, workerName }
   Response: { success, recordId, fields }

6. POST /api/optimize-route
   Purpose: Optimize delivery route
   Body: { stops: [{ id, address, lat, lng }, ...] }
   Response: { optimizedStops, duration, distance }

7. GET / (bonus)
   Purpose: Serve dashboard/landing page
   Response: HTML with API reference
