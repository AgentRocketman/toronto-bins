## Implementation Details

### **Sidebar Address Links**
- Changed from `<div>` to `<a>` tag
- URL format: `https://www.google.com/maps/dir/?api=1&destination=${latitude},${longitude}`
- Example: `https://www.google.com/maps/dir/?api=1&destination=43.6629,-79.3857`
- Styled as teal/green links with underline on hover
- Opens in new browser tab with `target="_blank"`

### **Map Tooltip Address Links**
- Map tooltips also have clickable addresses
- Same Google Maps directions format
- Styled to match the teal color scheme

## User Experience

**Workflow:**
1. Driver sees address in green/teal (signals clickable)
2. Driver clicks address
3. Google Maps opens in new tab
4. Google Maps uses driver's current location (device GPS)
5. Destination is pre-filled
6. Turn-by-turn navigation ready
7. Driver can follow directions or return to app

**Or via map:**
1. Driver clicks dot on map
2. Tooltip appears with address
3. Driver clicks address in tooltip
4. Google Maps opens same way
