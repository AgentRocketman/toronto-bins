# Session Update: v18 - Fixed Tooltip Address Decorations

## ✅ v18: REMOVED UNDERLINE & OVERLINE FROM TOOLTIP ADDRESSES

### Issue Reported:
- Map tooltip addresses had unwanted **underline** and **overline** decorations
- Made text appear messy and hard to read
- Looked like strikethrough text

### Root Cause:
- Google Maps InfoWindow applies default link styling
- CSS wasn't explicitly overriding text-decoration properties
- Browser default link styling was bleeding through

### Fix Applied:

**Added CSS rules:**
```css
.info-window-content h3 {
  text-decoration: none !important;
}

.info-window-content h3 a {
  text-decoration: none !important;
  color: #0f766e !important;
  border: none !important;
}

.info-window-content h3 a:visited {
  color: #0f766e !important;
}

.info-window-content h3 a:hover {
  text-decoration: none !important;
  color: #059669 !important;
}
```

**Updated inline styles:**
- Added `!important` flags to ensure no browser defaults override
- Removed any potential border styling
- Ensured consistent color across all states
