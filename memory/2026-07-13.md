### Agent Photo Extraction (PHP scraper fix)
- **Problem**: Original regex `!\[([^\]]*)\]\(([^)]+)\)\s*\n+\s*\[` matched the "Save for later" heart icon (`![Save for later](svg)`) instead of the actual agent photo
- **Root cause**: Agent photo uses `[![]](photo_url)](/agent/NUM/)` format — inline image (`![]`) inside a markdown link, NOT `![alt](url)`
- **Fix regex**: `'/\[!\[\]\(([^)]+)\)\]\(\/agent\/\d+\//'` — specifically targets the agent-link-wrapped photo pattern
- **Example URL**: `https://cdn.realtor.ca/individuals/TS638851890600000000/lowres/1168643.jpg`

## 🐛 Beds Regex Fix — "5+1" Format (2026-07-13 evening)

**Problem**: Listing 29783482 (252 Wright Ave) showed only "1" bed instead of "5+1". The markdown reads `5 + 1 Beds 4 Baths`. Old regex `/(\d+)\s+Beds?\b/` matched "5" then failed on `+` (not whitespace), then matched "1 Beds".

**Fix**: Updated regex to `/(\d+\s*\+\s*\d+|\d+)\s+Beds?\b/i` — captures `5 + 1` or `5+1` patterns. Applied same to baths.

## 🐛 Double-Fade Bug on End Card — Fixed

**Symptom**: End card appeared to darken/fade twice at end of video.

**Root cause**: The `barEndFade` timer in the overlay engine (`(totalFrames - 1) / fps - 1.0`) was calculated including end card frames, so it fired DURING the end card. The overlay code's bottom bar fade-out was overriding the end card's `alpha = 1` setting, causing the bottom bar to fade out mid-end-card (visible as brightness shift).

**Fix**: Added `if (isEndCard)` guard in bottom bar animation — skips fade-out and keeps `bottomBar.alpha = 1` during end card frames.

## 🐛 End Card Dim-Fade Bug — Fixed (2026-07-13 evening)

**Symptom**: End card dimmed down then brightened, like two slides overlapping (NOT the bottom bar double-fade, that was already fixed).

**Root cause**: Crossfade sets `endCard.alpha = transProgress` where `transProgress` goes 0 → `(tf-1)/tf` — never reaches exactly 1.0. When `isEndCard` starts, `endCard.alpha ≈ 0.95`. The guard `if (endCard.alpha < 1)` was true, so the fade-in code ran. On frame 0: `endT = 0`, `endCard.alpha = 0 * END_CARD_DUR / fadeIn = 0` → flashed to invisible, then re-faded in over 0.5s.

**Fix**: Changed condition to `endCard.alpha < 0.5` instead of `< 1`. If arriving from crossfade (already ~0.95), it just sets `alpha = 1` and skips the redundant fade-in. Fade-in only runs when end card starts fully cold.

**Files**: `public_html/agentado/index.html` — `isEndCard` block in render loop, line ~1270.
