### 18:09 UTC — Smart Routing by Distance Deployed

**1. ✅ BROWSER GEOLOCATION**
- Auto-detects driver's current location
- Uses device GPS (with fallback to Toronto center: 43.6629, -79.3957)
- Updates every 5 seconds
- Silently fails if permission denied

**2. ✅ PROXIMITY SORTING (Smart Order)**
- Roll-Ins shown first (📥 come back first)
- Then Roll-Outs (📤 bin drops)
- Within each group: sorted by distance (closest first)
- Uses Haversine formula for accurate Earth surface calculations

**3. ✅ SEQUENCE NUMBERS (1-8)**
- Green circle badges left of checkbox
- Auto-assigned based on driver proximity
- Updates dynamically as driver moves
- Visual reference for optimal route order

**4. ✅ UPDATED TORONTO ADDRESSES**
- All 8 addresses verified with real coordinates:
  - Roll-Ins: Bloor St E, Dundas St W, Roncesvalles Ave (2 locations)
  - Roll-Outs: Bloor St W, Bay St, King St W, Front St W

### Geolocation Implementation
- `navigator.geolocation.getCurrentPosition()` with timeout
- Falls back to Toronto center if denied/error
- Updates: Every 5 seconds via `setInterval()`
- No forced map recentering (respects driver view)

### Sequence Number Algorithm
```
1. Calculate distance from driver to each stop (Haversine)
2. Group by type (Roll-Ins vs Roll-Outs)
3. Sort each group by distance
4. Assign 1-8 sequence numbers
5. Render in order on sidebar
```
