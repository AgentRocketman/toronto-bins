## Street View Bug Fix — CRITICAL 🐛 ✅

### **Issue Found**
User reported Street View still not showing despite multiple deploys.

**Root cause:** Line 88 in `streetview.js`
```javascript
// ❌ WRONG
geocoder.geocode({ address: formattedAddress }, ...)  // undefined variable!

// ✅ FIXED
geocoder.geocode({ address: address }, ...)  // use parameter
```

The code was trying to geocode an **undefined variable**, so geocoding always failed → "address not found" error → Street View never rendered.

### **Impact**
- This was a typo/variable scoping bug introduced during last rewrite
- Explains why every test address returned "address not found" regardless of validity
- Single-line fix; now deployed

### **Verification Steps**
User should now test at: https://agentrocketman.com/index2.html
1. Enter Toronto address (e.g., "100 King St W, Toronto")
2. Click "Check my address"
3. Scroll down → Street View should appear

3. `/api/geocode.php` — Server-side geocoding proxy (Google Maps Geocoding API, key stays server-side, 1-year cache, biased to Toronto)
4. Airtable schema — Added `Lat` + `Lng` (number, precision 7) to Bookings (`tblKMhGnYjsH0z7Lj`) AND Orders (`tblGhNRi3ENwVpNty`)
5. Deployed full site to production
