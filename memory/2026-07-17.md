## ElevenLabs Audio Cache Key Bug — FIXED (Jul 17 ~03:00 UTC)

### The Bug
Chris reported new videos had audio from a previous listing. Root cause: `elevenlabs-tts.php` line 39 computed `$cacheKey = md5($phText . '|' . $voiceId)` BEFORE `$phText` was defined (line 52). `$phText` was null/empty, so **every listing got `md5('' . '|' . voiceId)`** — the same cache key. First audio generated was served for ALL subsequent listings.

### The Fix
Reordered the file so `$phText` (pronunciation fixes) is computed FIRST, THEN `$cacheKey` and `$cachePath` are computed from it, THEN the cache HIT check runs. Correct order:
1. Create cache directory
2. Compute `$phText` (ft→feet, sq ft→square feet, Etobicoke→Etobico)
3. Compute `$cacheKey = md5($phText . '|' . $voiceId)`
4. Check cache, return HIT if exists
5. Call ElevenLabs API if not cached

Also deleted stale cached files from the old wrong key.

## Photo Extension Bug — FIXED
- Photos from img-proxy URLs downloaded with `.php` extension (e.g., `photo_0.php`)
- Photo analyzer (both V1 and V2) filtered by image extensions only → found 0 photos → returned `{}`
- Fix applied in `prepare-3d-job.php`: after determining extension from URL path, force `jpg` if not a recognized image type
  ```php
  $ext = pathinfo(parse_url($url, PHP_URL_PATH), PATHINFO_EXTENSION) ?: 'jpg';
  if (!in_array(strtolower($ext), ['jpg', 'jpeg', 'png', 'webp', 'gif'])) $ext = 'jpg';
  ```

## STDERR Constant Bug in PHP-FPM — FIXED
- `subtitles-rewrite-v2.php` used `fwrite(STDERR, ...)` which only works in CLI
- When called via HTTP (curl), `STDERR` is undefined → PHP Fatal Error
- Fix: Replaced all `fwrite(STDERR, ...)` with `error_log()` calls (works in both CLI and FPM)
- Also installed `php8.3-mbstring` on VPS (required by subtitles-rewrite-v2.php)
- Photo analyzer used `fwrite(STDERR, ...)` too — also fixed to use `error_log()`
