## Bin Checkbox UI Ripped — bins now fully automatic ✅ 2026-06-24 03:17 UTC

Customer was right — checkboxes made checkout cumbersome, and the recurring path
let you check "Christmas Tree" on a weekly subscription (paradox).

### Changes
- `index.html`: removed all bin-selection UI + state + helpers (-84 lines). Booking
  payload now carries `binsByDate` (computed from selectedDates + Toronto schedule)
  for ad-hoc, and `calendarName` only for recurring. Exposed `window.CSV_DATA` so
  the JS file can read it.
- `airtable-bookings.js`: dropped `binsRecurring`. Recurring orders compute bins
  for the NEXT upcoming pickup date (today onward) using `window.CSV_DATA[calendarName]`.
  Every order (ad-hoc + recurring) also stores `Calendar Name` via `saveOrder()`,
  so the driver dashboard can recompute bins for ANY future date on demand.
- `service-routing.html`: untouched — already parses `Bins JSON` per order, so it
  Just Works.

### New Airtable columns (added via API)
- Bookings table `Calendar Name` — singleLineText — id `fldJfSyti34WAvtIS`
- Orders table `Calendar Name` — singleLineText — id `fldNcNrPtIbu2trwV`
